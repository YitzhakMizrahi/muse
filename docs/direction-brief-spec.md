# Direction Brief Spec

## Purpose

The Direction Brief is the core Muse v1 artifact.

It must be useful to:

- a solo founder trying to understand what their product should feel like
- a downstream design or build agent trying to execute the direction coherently

## Structure

### 1. Direction Name

Purpose:

- make the direction memorable
- make it easy to reference in comparison and handoff

Rules:

- short
- understandable
- memorable
- clear by default
- slightly brand-like only when that improves clarity or distinctiveness

### 2. Core Thesis

Purpose:

- summarize the direction in its most important strategic and aesthetic terms

Rules:

- 1 to 2 sentences
- preferably 1 strong sentence
- only use the second sentence if it adds meaningful strategic clarity

### 3. Who It Should Resonate With

Purpose:

- explain who is likely to respond positively to this direction
- justify the taste choices through audience logic

Rules:

- do not reduce this to demographics
- describe the kind of person and what they respond to

### 4. What It Should Feel Like

Purpose:

- define the intended emotional and perceptual qualities

Rules:

- 3 to 5 qualities
- concise
- written in professional, usable language

### 5. What It Should Not Feel Like

Purpose:

- define the boundaries that prevent execution drift

Rules:

- 3 to 5 avoided qualities or failure modes
- focused on preventing wrong execution

### 6. Brand Tone

Purpose:

- define how the product should sound

Rules:

- short prose, not just adjectives
- explain voice qualities, persuasion style, sentence behavior, and what kind of copy to avoid

### 7. Visual Principles

Purpose:

- define the core visual rules that should shape the execution

Rules:

- 3 to 5 concise design rules
- actionable
- specific enough to guide execution
- not descriptive fluff

### 8. Typography Direction

Purpose:

- define the roles and character of type in the system

Rules:

- describe display character
- describe body and interface character
- describe hierarchy behavior
- do not specify exact font families in v1

### 9. Color Logic

Purpose:

- define how color behaves in the system

Rules:

- describe base tone
- describe where contrast lives
- describe how accents are used
- describe what color should not do
- focus on behavior and hierarchy, not swatch naming alone

### 10. Layout / Composition Guidance

Purpose:

- define how space, emphasis, and reading order should work

Rules:

- describe reading path
- describe density versus openness
- describe hierarchy behavior
- focus on composition, not generic layout adjectives

### 11. Landing Page Narrative

Purpose:

- define the persuasion arc of the page

Rules:

- describe the story the page tells
- describe the order in which trust and desire are built
- do not reduce this to a section list

### 12. Section-by-Section Landing Page Outline

Purpose:

- make the direction operational for downstream execution

Rules:

- structured and concise
- define the purpose of each section
- define the content role of each section
- define how each section should feel

### 13. CTA Strategy

Purpose:

- define how the conversion ask should behave

Rules:

- describe the primary action
- describe the tone of the ask
- describe the level of urgency
- describe what the CTA should not sound like

### 14. Anti-Patterns

Purpose:

- prevent predictable execution failures

Rules:

- call out visual, tonal, or structural mistakes to avoid
- keep them specific enough to be operational

### 15. Why This Direction Is Recommended

Purpose:

- justify why Muse selected this option over the alternates

Rules:

- tie the recommendation back to strategic fit first
- then explain secondary advantages such as conversion potential or distinctiveness

### 16. Agent Handoff Prompt

Purpose:

- translate the brief into a builder-usable execution prompt

Rules:

- optimized for a design/build agent
- target a branded landing page optimized for signup conversion
- preserve the direction without turning into vague style instructions

## Notes

The recommended Direction Brief should be fully developed.

The two alternates should be lighter and faster to compare.
