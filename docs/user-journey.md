# First User Journey

## Goal

Define the first end-to-end workflow that proves the product.

## Primary User

A founder, designer, or creative developer with a rough idea and uneven references, but no fully formed creative direction.

## Starting Point

The user has one or more of:

- a product idea
- a short brief
- a few screenshots
- a mood or adjective set
- an audience they want to reach

They do not yet have a coherent direction.

## Core Flow

### 1. Enter The Spark

The user arrives on a cinematic landing surface and starts a session by providing:

- project title
- short concept description
- optional audience
- optional mood words
- optional visual references

### 2. Build Direction

The system synthesizes the input into a structured creative direction with:

- a core creative thesis
- 3 creative territories
- tone and voice guidance
- visual signals
- typography suggestions
- color mood suggestions
- interaction and motion cues

### 3. Compare Territories

The user reviews the three territories side by side.

Each territory should feel distinct rather than like minor wording variations.

### 4. Refine With Intent

The user chooses a direction and refines it through explicit actions such as:

- make it bolder
- make it calmer
- make it more premium
- make it more editorial
- make it feel more experimental

### 5. Save The Artifact

The resulting direction becomes a saved artifact the user can revisit, iterate, and eventually share.

## Experience Requirements

The first version must prove:

- input can be lightweight
- output feels designed and structured
- comparison is part of the workflow
- refinement is controlled by the user
- the result feels worth keeping

## First Release Scope

Include:

- landing page
- session creation flow
- generated territory view
- refinement controls
- saved session/artifact model

Do not include yet:

- collaboration
- public sharing
- complex auth
- team workspaces
- advanced asset generation
