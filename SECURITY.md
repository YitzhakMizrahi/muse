# Security Policy

## Reporting

If you discover a security issue in Muse, do not open a public issue with exploit details.

Report it privately to the repository maintainers with:

- affected area
- reproduction steps
- impact assessment
- any suggested mitigation

## Scope

Security concerns include:

- secret exposure
- auth or authorization flaws
- unsafe input handling
- dependency vulnerabilities with meaningful impact
- data leakage across user boundaries

## Response

The project should acknowledge valid reports, assess severity, and ship remediation in a documented patch.
